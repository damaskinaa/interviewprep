🆕 Новый скилл /diagnose
Для живых продуктов (как /market-research — для новых идей). Задаёт до ~15 вопросов и:
• челленджит твою цель — «зачем тебе это?», поднимается по графу твоих бизнес-работ и ищет точку роста уровнем выше;
• находит не одно горлышко, а ВСЕ риски и ВСЕ точки роста, которые видит методология;
• вытаскивает рискованные предположения в твоих текущих инициативах;
• показывает, что чинить первым, и направляет в нужный скилл.

🔧 Все 4 producer-скилла (market-research, craft-value-proposition, product-requirements, craft-go-to-market) получили 6 улучшений:
1. Helicopter-view в начале — что сделаем по шагам, где работает ИИ, где решаешь и валидируешь ты, чем отличаются Quick и Deep, сколько примерно времени и токенов.
2. Выбор формата вывода — Markdown или HTML. HTML легче читать: сворачиваемые блоки, рабочая навигация; ссылки кликабельны в обоих. Для тех, кому тяжело читать портянки markdown.
3. Критичное отношение к вводным — всё, что вы присылаете (лендинг, презентация, описание), теперь считается гипотезой, а не правдой. Скилл ищет риски внутри ваших данных и выносит их отдельным блоком: «вот что ты дал → вот какие риски я в этом вижу». Больше не зашивает непроверенную работу в стратегию молча.
4. Долг валидации — на первой странице артефакта видно, на скольких непроверенных допущениях он стоит (и сколько из них смертельных). Вердикт теперь «GO (to validation)», а не «GO» — чтобы не читалось как «строим 3 месяца».
5. Настраиваемый путь сохранения результата — под конвенции вашего репозитория.
6. Deep-режим перестал лениться — минимальный порог источников + самопроверка, плюс рекомендация подключить web-MCP (Firecrawl/Exa), если G2/Capterra блокируются.

Next Move Theory — это методология, которая даёт пошаговые алгоритмы для любых продуктовых решений. Она позволяет вам увидеть все тактические и стратегические ходы, которые вам доступны, и помогает выбрать наилучшую стратегию в вашей ситуации.

Next Move Theory — интегральная методология. Она объединила Advanced Jobs To Be Done, Юнит-Экономику, Riskiest Assumption Test, ABCDX-сегментацию, Теорию Ограничений и многие другие, чтобы дать возможность видеть все возможные стратегии.

Большое обновление по удобству, во многом по следам вашего фидбэка — спасибо! Главное: теперь должно быть понятнее, с какого скилла начать, и скиллы меньше используют терминологии методологий и ответы скиллов стали короче.

👉 Понятно, с чего начать. Каждый скилл (и сообщение после установки) теперь ведёт к /nmt-chat — это «входная дверь»: опиши ситуацию своими словами, и он сам подскажет, какой скилл запускать под твою задачу. Та самая боль «куда оно поставилось и с чего начать» — закрыта.

Long Story Short (обычным языком):
Zenith — это «автопилот-надстройка» над ИИ-агентами. Обычные инструменты типа Claude Code или Codex - это кабина, где ты сам сидишь и по шагам рулишь одним помощником: промпт → ответ, промпт → ответ. Zenith берёт готовые требования и сам запускает целую команду агентов (один планирует, другой пишет код, третий проверяет) и работает к цели часами без единого моего промпта. Подключается не через API с оплатой за токены, а прямо к обычной подписке Claude/ChatGPT — то есть кост фиксированный. По сути это одновременно и альтернатива Claude Code/Codex (другой «водитель»), и надстройка поверх них.


Long Story Short (техническим языком):
Zenith Harness (от Intelligent Internet) — это агентный harness/оркестратор: внешний слой, который держит agent loop, тулинг и память и раздаёт роли sub-агентам (planner / worker / reviewer) через MCP, гоняя длинные автономные миссии с self-review. Claude Code и Codex — тоже harness'ы, но заточены под интерактивный цикл «human-in-the-loop»; Zenith — под long-horizon multi-agent автономию. Работает поверх подписки (Claude Max / ChatGPT), а не pay-per-token API, поэтому потолок по стоимости фиксированный. Формально это и замена дефолтному harness'у (сам драйвит Opus 4.8 вместо Claude Code), и надстройка — оркестрирует агентов, каждый из которых под капотом дёргает ту же модель.
