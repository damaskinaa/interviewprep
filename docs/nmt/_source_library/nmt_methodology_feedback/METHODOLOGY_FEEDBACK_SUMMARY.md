# Methodology Feedback Summary

## User barrier

The methodology is valuable but the GitHub, English, install, terminal, and command environment can block non technical users.

## Workshop demand

Users may pay for a guided workshop that converts methodology into a concrete result.

## Codex compatibility

Codex uses $nmt style commands, not Claude slash command mental models.

## Harness feedback

NMT should run in a controlled harness, not a single prompt.

## Interview analysis feedback

Good interview analysis extracts Core Jobs, success criteria, alternatives, consideration set, existing solutions, problems, value hypotheses, and missing evidence.

## Missing evidence standard

The output must say what the data does not show.

## Implication for Codex

Every run needs manifest, source tags, evidence ledger, validation gate, and no carryover.
