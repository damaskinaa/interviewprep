Critical rule:

All prior research is DOC CLAIM unless verified by repo behavior, tests, field evidence, payment evidence, or reliable external source.

What this bundle is for:

1. Preserve the work already done.
2. Prevent duplicate documents from confusing Codex.
3. Let Codex read current product sources before old archived sources.
4. Keep market research separate from product facts.
5. Keep New Frontiers application material separate from product evidence.
6. Keep generic Hermes runs separate from NailIt truth.

Read first for product state, but still tag as DOC CLAIM.

2. active_research_context

Read for market, segment, RAT, and validation ideas. Do not treat as proof of demand.

Read for method patterns only.

Keep for history and contradiction checks. Do not use as current truth unless the run manifest allows it.

Use to avoid claiming the source library is fully archival.
