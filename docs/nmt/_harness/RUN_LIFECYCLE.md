# Run Lifecycle

## Stage 0 Workspace audit

Confirm working directory, git remote, root files, app code, tests, NMT skills, docs/nmt.

## Stage 1 Run creation

Create a run folder, run manifest, input folder, state folder, output folder.

## Stage 2 Source audit

List sources, classify each source, mark missing documents, identify duplicates.

## Stage 2A Methodology preservation gate

Read `METHODOLOGY_PRESERVATION_CONTRACT.md` and `METHODOLOGY_COVERAGE_LEDGER.md`.

Confirm the run preserves the original Next Move Theory core, preserves the Anastasia/team/book enhancement layer, marks unavailable methodology as `UNAVAILABLE_SOURCE`, and does not use NailIt as the methodology.

## Stage 3 Read only diagnosis

No edits. Identify product state, business state, evidence state, validation debt, source inventory, source classification, Advanced JTBD structure, Job Graph, Critical Chain, Segment + Job, value creation, RAT/RIT, Unit Economics where applicable, ABCDX where applicable, Theory of Constraints, subtraction, local vs global optimum, and field validation gates.

## Stage 4 Harness audit

Audit tests, scripts, commands, fixtures, external dependencies, reproducibility, user trust risks.

## Stage 5 Root cause investigation

If code defects block validation, identify root cause before writing tests.

## Stage 6 Tests only

Add tests before production patches.

## Stage 7 Narrow patch

Patch only proven root cause.

## Stage 8 Validation gate

End with validation debt, fatal if wrong, cheapest real world test, block or proceed.

## Stage 9 Field loop

Run real user, payment, trust, completion, contribution, and refusal tests.

## Stage 10 Application or investor truth loop

Convert evidence into claims without overstating proof.
