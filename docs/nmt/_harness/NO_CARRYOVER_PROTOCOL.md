# No Carryover Protocol

The purpose of this file is to prevent one business idea from contaminating another.

## Rule

A future run starts empty unless the run manifest imports sources.

Future business ideas must create their own:

1. docs/nmt/_source_library/projects/{project_name}/
2. docs/nmt/runs/{date}_{project_name}_{run_type}/

No NailIt source, segment, pricing, market size, channel, moat, validation conclusion, customer assumption, product assumption, or founder evidence may carry into a future run unless that future RUN_MANIFEST.md explicitly imports it as DOC CLAIM context.

## Forbidden carryover

Do not carry over:

1. NailIt segments
2. NailIt pricing
3. NailIt channels
4. NailIt market sizing
5. NailIt moat claims
6. NailIt competitor conclusions
7. NailIt New Frontiers framing
8. NailIt product state
9. NailIt validation conclusions
10. NailIt founder evidence

## Allowed reuse

The following may be reused for any idea:

1. NMT canon
2. Validation gate
3. Harness rules
4. Claim tagging schema
5. Interview analysis process
6. Evidence ledger template
7. Run manifest template
8. RAT logic
9. Subtraction before addition
10. GO means validation

## Import rule

If a future run needs NailIt as an analogy, it must say this in RUN_MANIFEST.md:

Imported project context:
docs/nmt/_source_library/projects/nailit/

Purpose:
Analogy only, DOC CLAIM, not evidence for this project.
